# Plano Oficial — Consolidação Cognitiva Enterprise (C0)

**IMPETUS · Fase C0 — Congelamento Arquitetural**  
**Data de vigência:** 2026-05-22  
**Status:** ATIVO

---

## 1. Declaração de intenção

O IMPETUS concluiu a construção do **Runtime Cognitivo multi-domínio (Z.18–Z.29)** e dos **7 cockpits native**. O problema principal deixou de ser falta de IA e passou a ser **fragmentação de autoridade cognitiva**.

A Fase C0 **congela expansão** para estabilizar autoridade, delivery, runtime, frontend, orchestration e governança.

## 3. Runtime oficial

```text
Runtime Cognitivo Z.18–Z.29
→ AUTORIDADE PRINCIPAL DO SISTEMA (cérebro oficial)
```

Inclui:

- Block registry Z.19 / Z.24
- Composition shadow + engine
- Render promotion controlada (Z.22)
- Cockpits native (quality, safety, hr, production, environmental, maintenance, executive)
- Adaptive orchestration Z.28 (recomendações only)
- Governance learning Z.29 (recomendações only)

---

## 4. Papel do Motor A

| Função | Descrição |
|--------|-----------|
| **Fallback** | Layout por cargo, KPIs legados, widgets legacy |
| **Degradation mode** | Quando runtime Z indisponível ou tenant sem promoção |
| **Safety runtime** | Rollback imediato sem mutation |
| **Rollback authority** | Única fonte garantida em incidente de delivery |

Motor A **não é aposentado** na C0/C1.

---

## 5. Papel do Engine V2

| Estado | Descrição |
|--------|-----------|
| **Experimental** | Motor contextual legado (Phase 2) |
| **Candidato à aposentadoria** | Auditoria comparativa obrigatória (C1) |
| **Regra** | Não expandir; medir redundância vs Runtime Z |

---

## 6. Estados oficiais de runtime

| Estado | Significado |
|--------|-------------|
| **OFF** | Desligado |
| **SHADOW** | Observando — compõe sem governar delivery |
| **ENRICH** | Influencia KPIs / insights sem governar widgets |
| **CONTROLLED** | Governa parcialmente via render promotion + cockpit native |
| **AUTHORITATIVE** | Governa delivery oficial (não activar na C0/C1) |

---

## 7. Regras obrigatórias (invioláveis)

- shadow-first
- additive-only
- rollback-safe
- deterministic governance
- anti-leakage
- semantic isolation
- supervised orchestration (Z.28)
- supervised learning (Z.29)

---

## 8. Proibições explícitas

NÃO activar:

- auto-remediation
- auto-decisions industriais
- autonomous shutdowns
- adaptive mutations automáticas
- authoritative mode global sem auditoria C1 completa
